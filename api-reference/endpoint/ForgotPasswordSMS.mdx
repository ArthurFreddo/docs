## Recuperar Senha via SMS

Este endpoint permite que os usuários solicitem a recuperação de senha através de uma mensagem SMS enviada para o número de telefone fornecido.

### Método HTTP

**GET**


---
### **URL** `/api/SMS`

### Cabeçalhos

| Header Name    | Tipo   | Descrição                                    | Obrigatório |
|----------------|--------|----------------------------------------------|-------------|
| `authorization`| string | Chave de autenticação da API no formato Basic | Sim         |

### Parâmetros da Query String

| Parâmetro     | Tipo    | Descrição                                              | Obrigatório |
|---------------|---------|--------------------------------------------------------|-------------|
| `phone`       | string  | Número de telefone do usuário, deve conter pelo menos 10 dígitos e apenas números. | Sim         |

### Exemplo de Requisição

```http
GET /api/forgotPasswordSMS?phone=99123456789 HTTP/1.1
Host: api.exemplo.com
Authorization: Basic <sua-chave-de-autenticação>
```

### Exemplo de Resposta com Sucesso

```json
{
  "error": false,
  "message": "SMS enviado com sucesso."
}
```

### Exemplo de Resposta com Falha de Validação

Se a validação falhar, por exemplo, com um número de telefone inválido, a resposta será:

```json
{
  "error": true,
  "message": "Validation failed",
  "details": [
    "O número de telefone deve conter apenas números.",
    "O número de telefone deve ter pelo menos 10 dígitos."
  ]
}
```

### Observações

- Caso o número de telefone seja inválido ou não esteja registrado no sistema, uma mensagem de erro será retornada.
- A requisição deve incluir o número de telefone correto no formato internacional, com pelo menos 10 dígitos.
