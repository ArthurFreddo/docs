## Recuperar Senha via E-mail

Este endpoint permite que os usuários solicitem a recuperação de senha através de uma mensagem enviada para o endereço de e-mail fornecido.

### Método HTTP

**GET**


---
### **URL** `/api/email`

### Cabeçalhos

| Header Name    | Tipo   | Descrição                                    | Obrigatório |
|----------------|--------|----------------------------------------------|-------------|
| `authorization`| string | Chave de autenticação da API no formato Basic | Sim         |

### Parâmetros da Query String

| Parâmetro     | Tipo    | Descrição                                              | Obrigatório |
|---------------|---------|--------------------------------------------------------|-------------|
| `mail`        | string  | Endereço de e-mail do usuário. Deve ser um e-mail válido. | Sim         |

### Exemplo de Requisição

```http
GET /api/forgotPasswordEmail?mail=usuario@email.com HTTP/1.1
Host: api.exemplo.com
Authorization: Basic <sua-chave-de-autenticação>
```

### Exemplo de Resposta com Sucesso

```json
{
  "error": false,
  "message": "E-mail enviado com sucesso."
}
```

### Exemplo de Resposta com Falha de Validação

Se a validação do e-mail falhar, como por exemplo ao inserir um e-mail inválido, a resposta será:

```json
{
  "error": true,
  "message": "Validation failed",
  "details": [
    "E-mail inválido. Por favor, insira um e-mail válido."
  ]
}
```


### Observações

- A requisição deve conter um endereço de e-mail válido. Se o e-mail não for encontrado no sistema, será retornada uma mensagem de erro.
- É importante que o formato do e-mail esteja correto para evitar falhas na validação.
